# Changelog

## 2026-06-19

### 修复
- **重启后重复回复**：重构已回复检测为配对模式 — 接收气泡下方右半区 `np.std()` > 30 即判定已回复，不再依赖绿色颜色匹配
- **PhotoImage GC 崩溃**：截图选择器由 tkinter 重写为 PySide6 QDialog（参照 MusicClassifier），消除 `image "pyimage1" doesn't exist` 错误
- **回复通知失效**：切换为 `winotify` + AUMID 快捷方式自动注册（`pywin32` + `WScript.Shell`），回复后弹出 Windows toast 通知
- **回复气泡框选错位**：`_analyze_region` 回复框坐标补回右半区偏移 `w // 2`

### 新增
- **启动延迟**：软件启动后 3 秒内不执行检测，TUI 底部显示黄色「启动中」
- **点击聚焦**：关键字匹配后点击消息气泡重新截图检测，防止输入区遮挡回复内容
- **回复通知**：回复成功后自动发送 Windows toast 通知
- **PySide6 截图选择器**：暗化遮罩 + 白色选边框 + 十字准星 + 确认/取消工具栏 + 自动 DPI 处理

### 优化
- **TUI 布局**：
  - 监控开关移至底部替代退出按钮，显示 `已启动`/`已停止`/`启动中`
  - 最后触发和回复用时合并为一行 `最后触发: 群名 @ 时间 用时 X 秒`
  - 去掉监控开关下方横隔线和 `(Enter 切换)` 提示
- **已回复检测**：从绿色颜色匹配 → 右半区背景排除 → `np.std` 配对检测，逐步简化至不可再简
- **依赖**：新增 `winotify>=1.1.0`、`PySide6`（已安装，参照 MusicClassifier）
- **清理**：删除全部过时测试文件和持久化历史残留代码
