# 2026-06-20

## Bug Fix: delay_min 无法保存

- **根因**: `ui/tui.py` `_commit_reply_edit()` 中跨字段校验 `delay_min > delay_max` 时直接 return 不保存。当 delay_max 为 0.0 时，任何正值的 delay_min 都被拦截。
- **修复**: 改为自动调整策略 — 编辑 delay_min 时如果新值超过 delay_max，自动将 delay_max 同步上来；反之亦然。约束始终保持，但不再阻止用户操作。
- **涉及文件**: `ui/tui.py` 第 397-412 行

## Bug Fix: 编辑模式按方向键显示 K/M/H/P 乱码（两次定位）

- **真正根因**: `msvcrt.getwch()` 在中文 Windows（代码页 CP936/GBK）下，扩展键前缀 `\xe0`（224）正好落入 GBK 双字节前导码范围（0x81-0xFE）。`_getwch()` 将其与紧随的扫描码合并为一个中文字符返回，导致 `"\xe0"` 字符串比对永远不触发，扫描码最终被 `isprintable()` 捕获加入编辑缓冲。
- **第一次尝试（失败）**: 轮询窗口替换 kbhit() — 无效，因为 `\xe0` 根本不会以独立字符形式到达 handler。
- **最终修复**: 编辑模式切换为 `msvcrt.getch()`（bytes），与非编辑模式的 `get_key()` 保持一致。扩展键前缀以 `b"\xe0"` 比对，扫描码直接 `getch()` 阻截。同时增加 GBK 双字节解码逻辑，保证中文输入正常。
- **涉及文件**: `ui/tui.py` 主循环第 66-69 行 + `_handle_reply_edit_char` 第 372-403 行
